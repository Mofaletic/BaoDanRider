# 视觉样板街标准

更新日期：2026-07-13  
状态：第一轮资产兼容性样板完成；核心工作日循环验收前冻结扩建，禁止批量铺设

## 目标

样板街用于证明一套美术语言能同时满足：

- 接近《胡闹厨房》的鲜艳、夸张、轻喜剧气质；
- 从车后第三人称镜头下能快速读懂道路、餐厅和住宅；
- 不改变已经验收的道路宽度、建筑占地和碰撞边界；
- 可拆成重复使用的模块，而不是每栋楼单独手工拼装；
- 在正式批量生产前暴露比例、枢轴、材质、碰撞和性能问题。

## 第一轮范围

- 位置：南侧起点向东行驶后，右侧的前两个建筑地块；
- 内容：一间餐厅、一栋住宅、两段人行道、餐车、灯和垃圾桶；
- 主素材包：`/Game/AnimeCitySuburbs`；
- `Cartoon_City_Free` 暂不作为本街区主建筑包。

## 资产审计结果

自动审计了 372 个静态模型：

- `AnimeCitySuburbs`：293 个，包含适合当前街区尺度的商店、住宅、人行道和街道道具；
- `Cartoon_City_Free`：79 个，主要建筑约 16–57 米高，更适合高楼城市，不适合当前紧凑配送街区。

## 碰撞规则

- 导入模型在样板中只负责画面，复杂碰撞全部关闭；
- 每栋楼使用 600 × 360 cm 的简单盒体碰撞，严格继承已验收白盒占地；
- 人行道和小道具暂时不阻挡车辆，避免再次出现不可见卡点；
- 正式美术替换不得改变道路和捷径的可通行宽度。

## 当前判断

第一轮证明了导入模型可以正确落地，比例与道路关系没有失控。但该素材整体偏灰、偏写实，与目标中的《胡闹厨房》式高饱和、夸张轮廓仍有明显距离。

因此当前建议是：

1. 保留 `AnimeCitySuburbs` 作为道具、结构参考和部分可复用模型来源；
2. 在样板街上增加统一的高饱和配色、粗轮廓、夸张招牌和更圆润的角色/车辆语言；
3. 风格样板通过玩家验收后，才允许扩展到其他建筑；
4. 核心特效只先做取餐、送达、碰撞和漂移四种，装饰性特效留到润色阶段。

## 当前冻结条件

2026-07-13 玩法方向新增了可见背包、停车下车、徒步进餐厅接单，以及限时工作日赚钱循环。样板街不再向其他地块扩展，直到以下白盒流程由玩家完整验收：

1. 骑车到餐厅、停车、下车并徒步进入接单区；
2. 接餐后背包容量与订单倒计时正确变化；
3. 回到车边上车、完成配送并无中断地获得下一单；
4. 工作日到时后正确处理未接订单或背包中的最后一单，并进入日终结算。
